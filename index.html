Lenguaje de programación

Antes de empezar a aprender sobre temas específicos de programación, conviene elegir un idioma de programación

C++

Lenguaje multiparadigma clásico, pero algo complicado de aprender

  • Usado para: desarrollo en general, de videojuegos, sistemas embebidos
  • Gestión de memoria manual
  • Para aquellos que quieren un reto al iniciar

C

Un Lenguaje de legado, pero seminal en la computación

  • Usado para: desarrollo en general, sistemas embebidos, IoT, soporte a sistemas de legado
  • Estructura y sintaxis sencilla
  • Para aquellos que quieren un aprendizaje desde lo más básico

Java

Lenguaje de uso general usado ampliamente en el sector laboral

  • Usado para: desarrollo en general, desarrollo web, desarrollo en pruebas
  • Multiparadigma
  • Buen lenguaje para aprender a programar y desarrollar proyectos útiles

Python

Lenguaje ampliamente utilizado. Popular por tener una estructura y sintaxis sencilla

  • Usado para: desarrollo en general, desarrollo web, ciencia de datos, y prácticamente en cualquier tarea donde se ocupe codificar
  • Multiparadigma
  • Para aquellos que quieran desarrollar proyectos útiles en producción lo más rápido posible

Go

Lenguaje desarrollado por Google, inspirado en C

  • Versión modernizada de C, con los features esperados de lenguajes modernos
  • Multiparadigma
  • Apto para cualquier programador

C#

Lenguaje propiedad de Microsoft. Usado ampliamente en ciertos campos

  • Usado para todo tipo de desarrollo, en particular, para Web (con ASP.NET)
  • Multiparadigma
  • Apto para cualquier programador

Rust

Lenguaje inspirado en C++, con la visión de ser un competidor moderno a ese lenguaje

  • Usado para todo tipo de desarrollo, en particular, para sistemas embebidos de IoT
  • Muy ligero y optimizado
  • Apto para aquellos interesados en sistemas embebidos (robots, dispositivos, sensores, etc.)

Estructuras de datos

Hace referencia al estudio de la gestión y manejo de datos hecho por la computadora. Los datos se deben de almacenar en memoria. Para esto, se usan estructuras que se pueden acceder y modificar cuando sea necesario.

Notación Asintótica

Hace referencia a la Notación matemática usada para definir el tiempo que toma a un algoritmo de ordenamiento en completarse, en terminos de 'n'. Esto se estudia al mismo tiempo que las estructuras de datos.

Arreglos

Se puede considerar la estructura de datos mas simple. Consiste en espacios de memoria reservados para almacenar una colección de valores

Lista de enlaces

Consiste de una estructura de datos que almacena información en nodos. Cada nodo consiste del dato en sí, y una referencia al siguiente elemento de la lista

Pila

Estructura de datos donde los datos que se van almacenando, se van empujando hacia el fondo de la pila. De tal forma que los últimos elementos en entrar, son los primeros en salir

Cola

Similar a la pila, excepto que aqui, los primeros datos en entrar, son los primeros en salir (similar a una fila común y corriente en la vida real).

Tabla Hash

Similar a la cola, excepto que aquí, todos los elementos van acompañados de un valor identificador, de tal forma que se almacenan datos en pares de Key/Value.

Arbol

Estructura donde los datos se almacenan en nodos. Los nodos pueden tener relación entre sí, de tal forma que forman una jerarquía de árbol.

Gráfico

Se refiere a un conjunto de estructuras representados en nodos y aristas. También conocidos como Grafos.

Notación Big-O

Notación matemática usada para representar los requerimientos de espacio y tiempo de un algoritmo, según la variable 'n' (cantidad de elementos)

Notación Big-Theta

Si la Notación Big-O y Big-Omega resultan ser iguales, entonces podemos darle una denominación de Theta.

Notación Big-Omega

Similar a Big-O, pero mide el resultado más optimista del Big-O.

Variaciones de arboles

Existen distintas estructuras en forma de árbol. Importante entre ellos, es el árbol binario. Un árbol binario tiene la característica de que los nodos pueden tener máximo dos hijos. Este tipo de estructura es popular debido a que es muy eficiente al momento de realizar búsquedas y operaciones recursivas.

Grafos

Otra estructura que consiste de nodos y cuenta con diferentes variaciones. Aunque los grafos pueden ser útiles, las estructuras tipo árbol son más relevantes.

Algoritmos de ordenamiento

Estos algoritmos son utilizados para ordenar elementos, comunmente de menor a mayor. Cada uno de estos tiene diferentes grados de complejidad; ademas de requerimientos de tiempo-espacio variables (medibles con la Notación de Big-O).

Tiempos de ejecución

Son los tipos de comportamientos que se presentan al ejecutar algoritmos, en términos de tiempo-espacio.

  • Constante: los recursos consumidos son iguales sin importar la cantidad datos. Caso ideal.
  • Logaritmico: los recursos siguen un comportamiento de log(n). Considerado muy eficiente.
  • Lineal: los recursos consumidos aumentan de forma directamente proporcional a la cantidad de datos a trabajar.
  • Exponencial: los recursos consumidos aumentan en factor de exponente. Considerado ineficiente.

Algoritmos de Grafos

Estos algoritmos son utilizados para distintas operaciones en estructuras de tipo grafo. Los más importantes son aquellos que se usan para búsquedas y recorridos.

Algoritmos de árboles

Estos algoritmos son utilizados para distintas operaciones en estructuras de tipo árbol.

Búsqueda de subcadenas

Estos algoritmos son utilizados para encontrar subgrupos dentro de un objeto. Por ejemplo, en un arreglo de caracteres (cadena), se usa para buscar una subcadena.

Recursión

Algoritmos basados en el principio de la recursión (la resolución de un problema en términos de si mismo).

Búsqueda

Algoritmos usados para encontrar elementos dentro de un conjunto.

  • Lineal: Se busca de forma secuencial, es decir, se recorre el conjunto de inicio a fin, deteniendose la ejecución cuando se encuentre el elemento
  • Binaria: Se busca dividiendo el conjunto en dos partes iguales, y realizando la búsqueda en el subconjunto que contenga el término a encontrar.

Algoritmos

Un algoritmo es una serie de pasos ordenados de tal forma que resuelven un problema. Aprender a pensar de forma algoritmica es una habilidad escencial para el desarrollo.

Algoritmos Codiciosos

Son aquellos que se enfocan a resolver el problema por partes; eligen cada parte dependiendo en el beneficio inmediato que ofrece la resolución de esa parte.

Algoritmos de Backtracking

Son aquellos que resuelven el problema por partes. El algoritmo trata de resolver cada subparte, y si falla, entonces se regresa para intentar resolverlo de otra forma.

Algoritmos de Cache

Son aquellos que se usan para gestionar la memoria cache en una computadora. Determinan cuales son los datos que se van a almacenar y eliminar del cache.

LRU

Este algoritmo elimina al dato que tiene mas tiempo sin usarse

LFU

Elimina al dato (almacenado en un par Key/Value) que es menos utilizado.

MFU

Es una variación de LRU. Elimina el elemento menos utilizado.

Operadores de Bit

Son operadores que se usan para manipular datos al nivel de bits. Se usan principalmente en criptografía, procesamiento de imagenes, etc.

Números flotantes

Son un tipo de dato numérico con la característica de que usan punto decimal. Se refiere a estudiar a detalle la composición y usos de los números de punto flotante.

Endianess

Se refiere al orden que toman los bytes al almacenarse en memoria.

Codificación de caracteres

Es la forma en la que se representan los caracteres alfanumericos para la computadora. El método más común es ASCII.

Patrones de diseño

Son buenas prácticas estándarizadas que ayudan al desarrollador a estructurar un software.

Patrones GoF

El Gang of Four (GoF) es un conjunto de 23 patrones descrito por un conjunto de 4 autores. Muy comunmente usados.

Patrones arquitectónicos

Son diseños de alto nivel, que estructuran un sistema completo. Se usa para evitar problemas comunes al desarrollar software.

Diagramas UML

UML (Unified Modeling Language) es una notación gráfica que consiste de símbolos estandarizados para hacer diagramas de sistemas. Estos diagramas son útiles en la etapa de diseño de sistemas, para describir el funcionamiento del software antes de iniciar.

Diagramas de clases

Diagrama UML usado para representar las clases dentro del código de un módulo o sistema. Aplica para sistemas desarrollados usando un paradigma orientado a objetos.

Diagramas de casos de uso

Diagramas que representan las acciones que un sistema va a manejar, ya sea con usuarios o con otros sistemas.

Diagramas de actividades

Diagrama que muestra el flujo entre acciones que un sistema realiza. Se usa para demostrar como se coordinan las actividades del sistema.

Diagramas de estado

Similar a los diagramas de clases, excepto que aquí se representan las clases en un tiempo de ejecución determinado (estado).

Diagramas de secuencia

Son diagramas de interacción que representan como y cuando se llevan a cabo operaciones. Muestran la interacción entre objetos en un sistema.

Habilidades matemáticas

Un dominio de temas básicos de álgebra, cálculo y probabilidad y estadística son convenientes al momento de programar.

Clases de complejidad

Se refiere a una clasificación de problemas aún no resueltos en el campo de la computación. Cada clase se compone de problemas de complejidad similar.

Tiempo Polinómico

Las clases de complejidad se dividen según el tiempo Polinómico que toma una máquina determinista en resolver un problema.

Tries

Son un tipo de estructura de datos que almacenan cadenas. Son una especie de árbol, donde cada nodo representa un caracter de la cadena.

Arboles de búsqueda balanceados

Son un tipo de árbol binario que se auto-balancean, lo que permite que operaciones de datos, como inserción, eliminación y busqueda, sean mucho más rápidas.

Diseño de sistema

Es el proceso de plantear la arquitectura, modulos, interfaces y datos de un sistema, para satisfacer los requerimientos especificados.

Bases de datos

Una base de datos es un conjunto de información útil almacenada. Se usan sistemas de gestión para poder manejar esta información.

SQL vs. NoSQL

Tradicionalmente, las bases de datos han sido en forma de tablas; estas tablas se pueden acceder y manipular con el uso de un lenguaje de consultas estructurado (SQL).

  • Mas reciente, ha sido el advenimiento del NoSQL, lenguajes y formas de almacenar bases de datos que no usan un modelo de tablas.

Redes

Una red es un conjunto de dos o más computadoras conectadas con el proposito de compartir información entre ellas. Buen conocimiento para tener en general, pero con mucho campo para especialización.

Seguridad

Se refiere a las acciones protectoras que los desarrolladores deben implementar para proteger aplicaciones web de posibles amenazas internas y externas.

Como funcionan las computadoras

Es muy útil entender como los circuitos eléctricos que conforman a una computadora son capaces de tomar información, procesarla, y darles salida.

Registros y RAM

La memoria RAM es el lugar donde se almacena información que el CPU esta proximo a utilizar. Todos los lenguajes de programación tienen metodos para manipular memoria y sus registros.

Procesos e hilos

Son las unidades más pequeñas de ejecución de un programa. Un proceso es una instancia de un programa que esta en ejecución.
